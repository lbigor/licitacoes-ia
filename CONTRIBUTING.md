# Contribuindo

Obrigado por querer melhorar este projeto.

## Fluxo

1. Abra uma issue descrevendo a mudança antes de codar.
2. Faça fork e crie branch `feat/<nome-curto>` ou `fix/<nome-curto>`.
3. Rode `./test.sh` localmente — tem que passar.
4. Abra PR contra `main` e preencha o template.
5. Aguarde revisão de @lbigor — merge é sempre manual.

## Regras

- Commits em português, imperativo: "adiciona X", "corrige Y".
- PRs pequenos > PRs gigantes.
- Testes obrigatórios pra qualquer mudança de código.
